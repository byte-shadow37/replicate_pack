Abstract

Refactoring is an essential practice in software maintenance that focuses on improving design and coding quality while fixing design flaws. The ‘Extract Method’ refactoring is especially popular for merging duplicate code segments into a single method. Several studies have investigated methods to recommend ‘Extract Method’ refactoring opportunities using techniques such as program slicing, program dependency graph analysis, change history analysis, structural similarity, and feature extraction. Yet, existing refactoring tools require developers to interrupt their workflow to address them. In this paper, we presents Clone Learning and Optimization with Neural Engines (CLONE), a novel feature of the IntelliJ IDEA plugin for just-in-time clone refactoring that instantly detects and suggest the refactoring of code clones, as soon as they are introduced by the developer, through the use of large language models (LLMs) of the developer’s choosing and a multi-agent architecture. The multi-agent system comprises a Detection Agent, a Refactoring Agent, a Compilation Agent, and a Testing Agent. The CLONE tool gives developers the freedom to use an intuitive, LLM-assisted, nonintrusive refactoring method that serves as an alternative to existing tools.
